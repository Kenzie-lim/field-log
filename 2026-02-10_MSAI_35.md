## Overview

### Context
1-hour ML class assignment requiring end-to-end pipeline: topic selection, data collection via open API, visualization & modeling, and organized deliverable. Builds on PCA, K-Means, and ensemble methods learned in recent weeks.

### Objective
Build a "Team Chemistry Simulator" — use Big Five personality data to find which personality combinations create the most diverse 6-person teams.

# 02-10 Web Scraping & Public API

One pattern runs through everything today: **request → parse → extract → store**.

- **Static scraping**: Requests + BeautifulSoup + CSS selectors → text or image
- **API**: Same HTTP request, but data comes pre-structured (XML/JSON) → skip the parsing mess
- **Key gotcha**: Encoding vs Decoding key — `requests.get(params=...)` auto-encodes, so use Decoding key to avoid double-encoding

Scraping is fragile and increasingly blocked. APIs are the clean path. Both use the same HTTP mechanics underneath.

## Project Flow

**1. Data Collection (Kaggle API)**
- What: Downloaded Big Five dataset (1M+ respondents × 50 Likert-scale items), filtered to 603K clean rows
- **Why**: Needed high-dimensional subjective evaluation data with enough volume for clustering
- **So what**: Kaggle API satisfies the "open API" requirement while giving a rich feature space (50 items across 5 personality dimensions)

**2. PCA → K-Means Clustering**
- What: PCA confirmed 5-component structure (46.5% variance explained). K-Means with K=6 selected via silhouette + cluster balance analysis
- **Why**: Reduce 50 features to interpretable dimensions, then segment people into actionable types
- **So what**: PCA independently validated Big Five theory from raw data. Silhouette scores plateaued at K=4-6 — people don't fall into discrete types, but 6 clusters gave the best balance between statistical quality and simulation richness (462 possible team combinations)

**3. Monte Carlo Team Simulation**
- What: Generated all 462 combinations_with_replacement(6 types, 6 people). Scored each by mean pairwise centroid distance in 5D PCA space
- **Why**: Test whether "one of each type" actually produces maximum diversity
- **So what**: It doesn't. Rank 1 doubles the EST extremes (Iron Wall × 2 + Sensitive Explorer × 2). "One of each" ranked only 5th. Diversity = magnitude of differences, not number of types.

**4. RandomForest Interpretation**
- What: RF regressor (R²=0.984) predicting diversity score from team composition. Feature importance identified which personality types drive diversity most
- **Why**: Needed a supervised model component + interpretable explanation of simulation results
- **So what**: Top 2 contributors (47.4% combined) are EST extremes. Bland middle-ground types contribute <10% each. The axis that matters most for team diversity is emotional stability — not extraversion or agreeableness.

---

## Key Insight
"Diversity isn't about collecting one of everything — it's about maximizing the distance between extremes." The intuitive assumption (balanced representation = maximum diversity) is mathematically wrong when cluster centroids aren't equidistant. Two bland types close together in PCA space contribute less than doubling a single extreme type.

## Deep Dive

- **Supervised vs. unsupervised pipeline clarity**: The core pipeline is PCA (unsupervised) → K-Means (unsupervised) → Monte Carlo (statistical simulation). RandomForest was added to provide a supervised modeling layer and interpretability — it predicts diversity from composition features, not the other way around. Understanding when to use each paradigm matters more than forcing one approach.

- **Feature importance source**: RF features are "count of each cluster type in team" (6 features), not PCA components. But PCA is the foundational infrastructure that makes meaningful clustering and diversity calculation possible. The pipeline layers build on each other: 50 items → 5D space → 6 types → team composition → diversity prediction.
