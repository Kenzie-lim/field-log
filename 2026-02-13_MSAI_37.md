# 02-14 Deep Learning: Perceptron to Backpropagation

## Overview

### Context
Builds on prior ML fundamentals (linear regression, gradient descent, loss functions). The course is structured so that each chapter solves a limitation of the previous one.

### Objective
Understand the full pipeline from perceptron → neural network → training via loss functions → backpropagation, and implement each step in Python.

---

## Lecture Flow

**1. Perceptron & Logic Gates (Ch01)**
- What: Implemented AND, OR, NAND, XOR gates using perceptrons with NumPy
- **So what**: XOR cannot be solved by a single perceptron → need for multi-layer networks. This is the origin story of deep learning.

**2. Neural Network Structure (Ch02)**
- What: Activation functions (step → sigmoid → ReLU), matrix operations, 3-layer neural network, softmax output, MNIST digit recognition with pre-trained weights
- **Why**: Adding nonlinear activation functions between layers gives the network the ability to learn complex patterns — stacking linear functions is still just linear
- **So what**: Softmax converts raw outputs to probabilities; the network can now classify, but weights are still manually provided. How to find them automatically?

**3. Training with Loss Functions & Numerical Gradient (Ch03)**
- What: Implemented MSE and cross-entropy loss, numerical differentiation, gradient descent, `TwoLayerNet` class, mini-batch SGD training loop
- **Why**: Loss function measures "how wrong" the network is. Gradient tells "which direction to adjust weights." Together they enable automated learning.
- **So what**: Numerical gradient works but is extremely slow — requires full forward pass per weight. Need a faster method.

**4. Backpropagation (Ch04)**
- What: Chain rule, addition/multiplication node backprop, Affine/Softmax-with-Loss layers, `TwoLayerNet` with `gradient()` method using backprop
- **Why**: Backprop computes all gradients in one backward pass instead of per-weight forward passes
- **So what**: Training accuracy jumped from ~10% (random) to ~90% in 1000 iterations. The full learning pipeline is now complete: forward → loss → backward → update → repeat.

---

## Key Insight
Every chapter solves the previous chapter's problem: single perceptron can't do XOR → multi-layer network. Network can't learn → loss function + gradient. Numerical gradient is too slow → backpropagation. The entire deep learning stack is a chain of problem-solution pairs.

---

## Deep Dive

- **w, b pairs define behavior**: AND, OR, NAND are identical structures — only weights and biases differ. There is no single correct pair; any (w, b) that satisfies the truth table works. This is why different initializations lead to different solutions.

- **Cross-entropy = -log(probability at correct class)**: Low confidence on the correct answer → large penalty. The name sounds complex but the operation is just -log.

- **Numerical differentiation vs. partial derivative**: Partial derivative = "which variable to differentiate" (what). Numerical differentiation = "approximate by nudging" (how). The code does both simultaneously — partial derivatives computed via numerical method.

- **"Differentiate c with respect to a"**: Means "how much does c change when you nudge a?" Visualize as: a on x-axis, c on y-axis, find the slope.

- **Chain rule as propagation of influence**: If a → t → c, then a's influence on c = (a's influence on t) × (t's influence on c). Backprop traces this influence chain from output back to input.

- **Vanishing gradient**: When backprop multiplies through sigmoid layers, gradients shrink exponentially toward input layers. Solutions: ReLU (ch05), He initialization (ch05), ResNet shortcuts (ch07).

- **`__pycache__` pitfall**: Python caches compiled .pyc files. When editing .py source files, must delete `__pycache__/` and restart kernel to pick up changes.
